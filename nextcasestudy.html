Case Study

  • Research
    Research
    User Research

    Through a series of interviews, I was able to gather information about what the client goals are when booking a flight through an online website. Overall, the users like the many ways they can search for flight but all felt the layout was cluttered and overwhelming to look at.

    User Goal

    The user wants a simple to read interface that allows them to compare prices to a set of personal criteria and weigh the cost to any given flight option.

  • Low Fidelity Sketch
    Low-fidelity Sketch

    These low fidelity wireframe sketches were used to iterate different versions for the website layout.


  • Sketch Wireframe
    Sketch Wireframe

    The wireframe sketches were then digitized in Sketch.

  • Version 1 Prototype
    Version 1 Prototype

    Version One Axure Prototype
  • Test
    Test

    A small sample of users were used to test the "Flight Finder" prototype concept. A report was created outlining the issues users had and the recommendations for improvement.

    Final Prototype

    Based on user findings, a revised prototype was created.
